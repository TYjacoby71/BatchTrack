# TEMP API Standardization Checklist

## Instructions
- Goal: standardize API response systems and emergency fallback behavior before implementation starts.
- Scope A (API Systems): every backend response surface currently returning JSON/errors/HTTP status patterns must move to one consistent API contract and helper path.
- Scope B (Emergency Fallbacks): native dialog and popup fallbacks are allowed only for hard-degradation scenarios and should be removed/reduced where shared helpers can fully cover.
- Status meanings: `incorrect` = inventory complete, pending implementation; `done` = standardized and validated in implementation pass.
- Priority meanings: `P1` = global/public/high-risk, `P2` = core feature flows, `P3` = lower-risk/supporting surfaces.

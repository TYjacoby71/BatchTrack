Redis provisioning & presets:
  • Before launch, create a managed Redis instance (Render Redis, Upstash, AWS ElastiCache) and set REDIS_URL once. The app mirrors it into RATELIMIT_STORAGE_URL automatically unless you override it.
  • Use SESSION_REDIS_MAX_CONNECTIONS to cap each worker’s connection pool (default 10). Match this to your Redis plan’s maxclients.
  • Set WEB_CONCURRENCY only as high as the Redis/Postgres plans can handle. During testing, keep it at 1–2; raise both concurrency and Redis limits together when upgrading.
  • For high-usage tiers (10k+ active users), a single well-sized Redis node normally suffices—scale memory and maxclients before considering sharding.
The app refuses to start in production without Redis-backed sessions, caching, and rate limiting.

Domain Event Dispatcher

      

        The app writes every significant action into the domain_event outbox table. To push those events downstream, run the dispatcher alongside your web workers.
      

      
    
        
  1. Provision any HTTP endpoint that should receive events (or leave the webhook unset to mark events processed without delivery).
    2. 
        
  2. Set DOMAIN_EVENT_WEBHOOK_URL=https://your-endpoint (optional if you only want local logging).
    3. 
        
  3. Run the worker: flask dispatch-domain-events (use --once for batches, or keep it running).
    4. 
        
  4. Monitor logs for retries; events exceeding the retry limit are marked with _dispatch_errors.

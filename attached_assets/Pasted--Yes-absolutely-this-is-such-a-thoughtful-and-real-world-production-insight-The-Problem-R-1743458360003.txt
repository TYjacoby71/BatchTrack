🧠 The Problem:
Right now, “tags” and “notes” are entered only when starting a batch.
But life doesn’t always work like that.

Your wife makes a batch → forgets to write notes

A week later someone says, “that balm was amazing!”

She wants to write: "Client LOVED this. Make again."

Or tag it: "customer-favorite" or "holiday-2025"

✅ The Solution:
Allow editing tags and notes after the batch is created — from the batch list or a batch detail page.

✅ Here’s What This Adds:
Feature	Description
📝 /batches/<batch_id>/edit-notes	Edit notes and tags for a specific batch
✅ Form with prefilled values	Show current notes + tags
💾 Saves back to data.json	Updates batch entry
🔗 Add “Edit Notes” link on /batches page	One-click access
🔧 CODE TO ADD — Copy & Paste Instructions

@batches_bp.route('/batches/<batch_id>/edit-notes', methods=['GET', 'POST'])
def edit_batch_notes(batch_id):
    data = load_data()
    batch = next((b for b in data['batches'] if b['id'] == batch_id), None)

    if not batch:
        return "Batch not found", 404

    if request.method == 'POST':
        batch['notes'] = request.form.get("notes", "").strip()
        batch['tags'] = [t.strip().lower() for t in request.form.get("tags", "").split(",") if t.strip()]
        save_data(data)
        return redirect('/batches')

    return render_template("edit_batch_notes.html", batch=batch)
